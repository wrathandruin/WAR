# Wrath and Ruin - Feature Delivery Map

This document organizes features by **delivery horizon**, not by wishlist breadth.

The goal is to keep the project aligned with an attainable route to alpha, beta, and market.

---

## Tier 0 - Production Foundations

These are not optional. They are what make the rest of the roadmap shippable.

- reproducible builds
- runtime data boundaries
- authoritative server architecture
- shared simulation layer
- dedicated server packaging
- local demo checklist
- staging and deployment plan
- telemetry and crash capture
- admin and support tools
- content and asset source-of-truth rules
- content validation pipeline
- installer and update strategy planning

---

## Tier 1 - Alpha Must-Haves

These define the first credible playable product slice.

### Core Play

- top-down point-and-click movement
- inspect and contextual interaction
- six-second combat loop
- inventory and equipment
- survival hazards
- persistent terrain consequences
- one complete mission chain

### World

- one planetary location
- one local space or orbital layer
- one second destination
- one ship class
- one authored encounter set

### Product Surfaces

- hosted authoritative session
- persistent world state
- internal tester onboarding flow
- playable UI shell
- debug overlays and diagnostics
- repeatable server deploy path

### Identity

- preserve the MUD soul through text, logs, speech, emotes, inspect text, and narrative output

---

## Tier 2 - Beta Should-Haves

These expand reliability, usability, and testability after alpha.

- broader onboarding and tutorialization
- accessibility pass
- stronger inventory, mission, and combat UI
- external tester support flow
- moderation and admin workflow
- content authoring acceleration
- progression and economy-lite pass
- more destinations and encounter variety
- better reconnect and session continuity
- launcher or updater path if required for outside testing

---

## Tier 3 - Market Readiness

These are the features and systems required to credibly take the project to market.

- commercial packaging and installer
- update and patch lane
- release candidate and rollback flow
- analytics and health monitoring
- crash reporting
- first-session polish
- live-ops tooling
- support and incident response workflow
- compliance and policy readiness for the chosen platform
- store or distribution readiness

---

## Deferred Until After Alpha

These may be valuable, but they should not be allowed to dilute the alpha slice.

- broad faction campaign coverage
- large planetary count
- deep construction breadth
- deep crafting breadth
- large economy simulation
- wide ship roster
- large-scale social governance systems
- full cross-platform launch support

---

## Legacy Features Worth Preserving Conceptually

These come from the older MUD lineage and should inform the new product even when the implementation changes.

- text-first roleplay surfaces
- persistence and world continuity
- social identity and speech systems
- contracts and mission framing
- faction and reputation potential
- operational rigor around deploys and runtime safety

---

## Working Rule

If a feature does not help:

- strengthen the production base
- complete the alpha slice
- improve beta readiness
- or improve market readiness

then it is not urgent right now.
